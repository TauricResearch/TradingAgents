# 📊 Codebase Guide: TradingAgents *(quick — 2 min)*

_Generated by codebase-guide skill on 2026-06-05._

## What it does
**TradingAgents** is a **multi-agent LLM financial trading framework** that simulates a real trading firm. Specialized LLM agents (analysts, researchers, trader, risk managers) collaborate and debate to produce a BUY/SELL/HOLD decision for a given ticker and date. It's a research project (paper: arXiv 2412.20138), v0.2.5.

## Project structure
```
tradingagents/            # core package
├── agents/
│   ├── analysts/         # fundamentals, market, news, sentiment, social_media
│   ├── researchers/      # bull_researcher, bear_researcher (debate)
│   ├── managers/         # research_manager, portfolio_manager
│   ├── risk_mgmt/        # aggressive / conservative / neutral debators
│   ├── trader/           # trader.py
│   ├── utils/            # agent_states, memory, data tools, structured output
│   └── schemas.py
├── graph/                # LangGraph wiring
│   ├── trading_graph.py      # ← main orchestrator / entry
│   ├── setup.py, propagation.py, conditional_logic.py
│   ├── reflection.py, signal_processing.py, checkpointer.py
├── dataflows/            # data providers (alpha_vantage*, y_finance, reddit, stocktwits)
└── llm_clients/          # provider factory, capabilities, model_catalog, validators
cli/                      # typer CLI app (main.py, config, models, stats)
scripts/                  # report_workflow.py, run_one.py, batch runners
tests/                    # unit / integration / smoke (pytest markers)
```

## The agent pipeline (mirrors a trading firm)
```
Analyst Team ─→ Researcher debate ─→ Research Mgr ─→ Trader ─→ Risk debate ─→ Portfolio Mgr ─→ Decision
(fundamentals,   (bull vs bear)                              (aggressive/                    (BUY/SELL/
 sentiment,                                                   conservative/                   HOLD)
 news, technical)                                             neutral)
```

## How to run
- **CLI:** `tradingagents` (Typer app at `cli/main.py`)
- **Reports refresh:** `python scripts/report_workflow.py --analysis-date YYYYMMDD` (per CLAUDE.md — don't hand-edit generated docs)
- **Tests:** `pytest` (markers: `unit`, `integration`, `smoke`)
- **Docker:** `docker-compose up`
