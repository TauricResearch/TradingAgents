# Design Document: TradingAgents Dashboard

## 1. Objective
Provide a real-time, visually intuitive interface to monitor the multi-agent trading pipeline, inspect agent reasoning, and track portfolio performance.

## 2. Architecture
- **Frontend:** React 18, TypeScript, Vanilla CSS.
- **Visuals:** `reactflow` for DAG visualization.
- **Backend:** FastAPI (Python 3.12).
- **Data Source:** Shared filesystem (PVC) containing `.json` logs and memory logs.

## 3. UI Components
### A. Agentic Flow Graph (Main View)
- Nodes represent specific agents (e.g., Market Analyst, Bear Researcher).
- Edges represent state transitions.
- **Interaction:** Clicking a node opens a side-drawer with:
    - Input parameters.
    - Raw tool outputs.
    - Agent's "Thought" process (LLM response).

### B. Trade History Sidebar
- List of recent tickers analyzed.
- Outcome indicators (Win/Loss/Pending) based on the `memory_log`.
- Links to "Reflections" generated by the system.

### C. Performance Dashboard
- Total Alpha vs. Benchmark (SPY).
- Hit rate and average holding period.

## 4. API Endpoints
- `GET /api/runs`: List all historical agent runs.
- `GET /api/runs/{id}`: Detailed state of a specific run.
- `GET /api/stats`: Aggregate performance metrics from memory logs.
- `GET /api/status`: Check if a CronJob is currently active.

## 5. Visual Aesthetic
- **Theme:** "Bloomberg Terminal meets Modern IDE."
- **Colors:** Deep charcoal backgrounds, emerald for "Bullish," ruby for "Bearish," and pulsing gold for active agent nodes.
- **Typography:** Inter for UI, Fira Code for agent logs.
