# ADR-001: Fix Notion File Upload — HTTP 400

**Status:** Proposed  
**Date:** 2026-04-27  
**Author:** William

---

## Context

Code upload file lên Notion đang trả HTTP 400. Module hiện tại sử dụng sai URL endpoint và sai Content-Type header cho multipart upload.

## Root Cause Analysis

| # | Bug | Code hiện tại | Đúng |
|---|-----|--------------|------|
| 1 | Sai URL tạo file upload | `https://notion.com` | `https://api.notion.com/v1/file_uploads` |
| 2 | Sai URL send file | `https://notion.com/{id}/send` | `https://api.notion.com/v1/file_uploads/{id}/send` |
| 3 | Sai Content-Type khi send | Gửi `application/json` header | Endpoint send cần `multipart/form-data` — **không được truyền Content-Type header**, để `requests` tự set |
| 4 | Không check response status | Không check `.status_code`, không `.raise_for_status()` | Cần check mỗi step, raise nếu fail |

## Decision — Corrected Implementation

```python
import logging
from pathlib import Path

import requests

logger = logging.getLogger(__name__)

NOTION_TOKEN = "your_integration_token"
PAGE_ID = "your_page_id"
FILE_PATH = Path("document.pdf")

# Base headers — JSON endpoints
_JSON_HEADERS = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Content-Type": "application/json",
    "Notion-Version": "2026-03-11",
}

# Headers for multipart send — NO Content-Type (requests sets it with boundary)
_UPLOAD_HEADERS = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Notion-Version": "2026-03-11",
}

BASE_URL = "https://api.notion.com/v1"


def upload_file_to_notion(page_id: str, file_path: Path) -> str:
    """Upload a file to Notion and attach it to a page as cover.

    Returns:
        The file upload ID.

    Raises:
        requests.HTTPError: On any non-2xx response.
        FileNotFoundError: If file_path doesn't exist.
    """
    if not file_path.exists():
        raise FileNotFoundError(f"File not found: {file_path}")

    content_type = _guess_content_type(file_path)

    # Step 1: Create file upload object
    create_resp = requests.post(
        f"{BASE_URL}/file_uploads",
        headers=_JSON_HEADERS,
        json={
            "mode": "single_part",
            "filename": file_path.name,
            "content_type": content_type,
        },
        timeout=30,
    )
    create_resp.raise_for_status()
    upload_id = create_resp.json()["id"]
    logger.info("Created file upload: %s", upload_id)

    # Step 2: Send file content (multipart/form-data — NO Content-Type header)
    with open(file_path, "rb") as f:
        send_resp = requests.post(
            f"{BASE_URL}/file_uploads/{upload_id}/send",
            headers=_UPLOAD_HEADERS,  # NO Content-Type here
            files={"file": (file_path.name, f, content_type)},
            timeout=120,
        )
    send_resp.raise_for_status()
    logger.info("File sent successfully for upload: %s", upload_id)

    # Step 3: Attach to page as cover
    patch_resp = requests.patch(
        f"{BASE_URL}/pages/{page_id}",
        headers=_JSON_HEADERS,
        json={
            "cover": {
                "type": "file_upload",
                "file_upload": {"id": upload_id},
            }
        },
        timeout=30,
    )
    patch_resp.raise_for_status()
    logger.info("Attached upload %s as cover to page %s", upload_id, page_id)

    return upload_id


def _guess_content_type(file_path: Path) -> str:
    """Map file extension to MIME type."""
    _MIME_MAP = {
        ".pdf": "application/pdf",
        ".png": "image/png",
        ".jpg": "image/jpeg",
        ".jpeg": "image/jpeg",
        ".gif": "image/gif",
        ".webp": "image/webp",
    }
    return _MIME_MAP.get(file_path.suffix.lower(), "application/octet-stream")
```

## Constraints

- File phải được attach trong **1 giờ** sau upload, nếu không Notion tự archive.
- `single_part` mode giới hạn **20 MB**. File lớn hơn cần `multi_part` mode (chỉ paid workspace).
- Tên file tối đa **900 bytes**.

## Debate

**Challenger:** Code trên chỉ attach cover. Nếu user muốn attach file vào block content thì sao? Và không có retry logic cho rate limit 429.

**Defender:** Scope hiện tại là fix 400 error — chỉ cần đúng URL + headers. Attach vào block content là feature riêng, dùng `PATCH /v1/blocks/{id}/children` với `file` block type. Retry logic nên dùng `tenacity` hoặc `urllib3.Retry` — nhưng đó là enhancement, không phải root cause.

**Verdict:** Fix 4 bugs trên là đủ để hết 400. Retry + block attachment là follow-up ticket.

# Code Review: `tradingagents/graph/transform.py`

**Date:** 2026-04-27  
**Reviewer:** Claude  
**Verdict:** Request Changes

## Summary

Module transform `final_state` thành `ChartJSON` qua GPT-5.3-Codex-High. Pydantic schema đã tốt, retry logic OK. **Vấn đề chính: prompt quá lỏng, LLM không nhất quán khi output JSON** — đặc biệt thiếu structured output enforcement.

## Critical Issues

| # | Line | Issue | Severity |
|---|------|-------|----------|
| 1 | 241-247 | System prompt quá generic, không có explicit JSON schema definition | 🔴 Critical |
| 2 | 145-160 | User prompt chỉ show example + free-text rules → LLM hay hallucinate field names, miss reference lines | 🔴 Critical |
| 3 | 256-261 | Không dùng `structured output` / `response_format` của OpenAI → LLM trả text lẫn JSON | 🟡 Major |
| 4 | 48-49 | `PriceHistory` validate `min_length=1` cho cả `dates` và `prices` nhưng không validate chúng **cùng length** | 🟡 Major |

## Recommended Fixes

### Fix 1: PriceHistory validator — cùng length dates/prices

```python
from pydantic import model_validator

class PriceHistory(BaseModel):
    """Historical price data points."""
    dates: list[str] = Field(min_length=1)
    prices: list[float] = Field(min_length=1)

    @model_validator(mode="after")
    def _dates_prices_same_length(self) -> "PriceHistory":
        if len(self.dates) != len(self.prices):
            raise ValueError(
                f"dates ({len(self.dates)}) and prices ({len(self.prices)}) "
                f"must have equal length"
            )
        return self
```

### Fix 2: System prompt — embed JSON schema thay vì example

```python
_SYSTEM_PROMPT = """You are a financial data extraction engine.
Extract trading plan data into the EXACT JSON schema below.

JSON SCHEMA (follow field names, types, and nesting EXACTLY):
{schema}

RULES:
1. Output ONLY valid JSON. No markdown, no explanation.
2. Every field is REQUIRED unless marked optional.
3. reference_lines: extract ALL price levels from the trading decision.
   Color mapping:
   - Take Profit (TP) → "green"
   - Entry → "orange"
   - Breakout/Add → "blue"
   - Stop Loss (SL) → "darkred"
   - Short entry/TP → "red"
   - Caution/Reduce → "darkorange"
4. price_history.dates and price_history.prices must be same length.
5. y_axis_range: [lowest_reference_line - 5%, highest_reference_line + 5%]
"""
```

### Fix 3: Dùng OpenAI structured output (`response_format`)

```python
json_schema = ChartJSON.model_json_schema()

system_prompt = _SYSTEM_PROMPT.format(
    schema=json.dumps(json_schema, indent=2)
)

response = llm.invoke(
    messages,
    response_format={
        "type": "json_schema",
        "json_schema": {
            "name": "ChartJSON",
            "strict": True,
            "schema": json_schema,
        },
    },
)
```

## What Looks Good

- Pydantic schema cho `ChartJSON` rõ ràng, match `transform_flow.json`.
- Retry logic với exponential backoff handle transient errors.
- Markdown fence stripping trong `_parse_llm_response`.
- Test coverage tốt, mock đúng I/O boundary.

## Debate

**Challenger:** `response_format` chỉ work với OpenAI API. Nếu dùng Anthropic hoặc local model thì sao? Và việc embed full JSON schema vào prompt tăng token count.

**Defender:** Module này hardcode `gpt-5.3-codex-high` (OpenAI) — nên `response_format` luôn available. Token cost cho schema (~200 tokens) negligible so với toàn bộ final_state dump. Nếu sau này mở rộng sang provider khác, fallback về prompt-only mode.

**Verdict:** Dùng `response_format` + embedded schema cho OpenAI path. Giữ prompt-only fallback nếu `response_format` raise `NotImplementedError`.
